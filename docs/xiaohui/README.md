# 🏢 小辉数字员工 2.0 (DevLoop 的母系统)

> DevLoop 的"出身背景"介绍 — 想知道 DevLoop 从哪儿来,看这里。

---

## 📖 是什么?

**小辉数字员工 2.0** 是一个**集成飞书/企微/微信多通道的企业数字员工系统**。

简单说: 一个能自动干活 (回消息 / 处理订单 / 管库存 / 写文档) 的"AI 员工"系统,装进企业飞书/企微就能用。

---

## 🎯 能做什么?

围绕企业的日常运营,自动完成:

| 模块 | 能力 |
|---|---|
| 📱 **消息接入** | 飞书 / 企微 / 微信 / 后台 4 通道统一接入 |
| 🛒 **商城管理** | 商品 / 订单 / 支付 / 配送 / 售后 全流程 |
| 👥 **CRM** | 客户 / 销售线索 / 跟进 / 合同 |
| 📦 **库存 + 物流** | 多仓 / 调拨 / 盘点 / 库龄 / 配送 |
| 💰 **财务** | 应收 / 应付 / 报表 / 发票 / OCR |
| 🏭 **采购 + 质检** | 采购单 / 合同 / 质检流程 |
| ✅ **审批** | 审批流 / 实例 / 委托 / 升级 |
| 🖨️ **打印机 + 售后** | 打印控件 / 售后维修 / 智能派单 |
| 📊 **数据分析** | 销售分析 / 报表 / 仪表盘 |
| 🤖 **AI 工作伴侣** | 对话购物 / 智能助手 |

---

## 💡 有什么优点?

### 1️⃣ **多通道集成**
- 飞书 / 企微 / 微信 / 后台 4 通道统一接入
- 一个系统,多个入口

### 2️⃣ **开箱即用**
- 22 个后端模块 + 152 张数据库表 + 27 个前端视图
- Docker Compose 一键部署
- 前后端分离 (Vue 3 + Fastify)

### 3️⃣ **AI 原生**
- OpenClaw 大脑 + MiniMax / DeepSeek LLM
- AI 能力浸润到每个模块 (对话购物 / 智能派单 / OCR / ASR)
- **DevLoop (开发任务调度中枢) 是它的 AI 中枢**

### 4️⃣ **完整 ERP / CRM**
- 不是玩具,是企业级系统
- 商品 / 订单 / 财务 / 库存 / 采购 / 物流 / 售后 全打通

### 5️⃣ **开源 + 私有化**
- MIT 开源协议 (跟 DevLoop 一致)
- 支持私有化部署
- 数据自己掌控

## 🔗 DevLoop 与小辉的关系

```
┌─────────────────────────────────────────────┐
│         小辉数字员工 2.0 (母系统)            │
│   22 模块 · 152 表 · 飞书/企微/微信集成      │
│                                             │
│   ┌──────────────────────────────────────┐ │
│   │ DevLoop (开发任务调度中枢 · 8 角色)    │ │
│   │     👉  剥离 → 开源 = DevLoop        │ │
│   └──────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

- ✅ **DevLoop 拿走的**: 数据底座 + 智能体调度 (0 业务耦合)
- ❌ **DevLoop 不包含**: 小辉的 CRM / ERP / 商城 等业务模块
- 🔄 **DevLoop 反哺**: 用 DevLoop 调度开发小辉自身,反过来提升小辉迭代速度
