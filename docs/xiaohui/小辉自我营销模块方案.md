# 小辉自我营销模块 — 方案设计

> 版本: V2.1 | 日期: 2026-05-21 | 状态: 方案阶段（双系统架构：系统推广 + 商城增长）

---

## 零、产品定位：两个营销引擎

自我营销不是"一个功能"，是**两个独立但协同的增长引擎**。

### 引擎一：小辉系统推广

**卖的是小辉数字员工系统本身。**
目标客户：需要 ERP/CRM 的中小企业、批发商、商贸城商户。
手段：内容引流 → AI 演示 → 成交/试用开通。

| 维度 | 说明 |
|------|------|
| 产品 | 小辉数字员工系统（SaaS） |
| 目标用户 | 中小企业老板、批发商、商贸城商户 |
| 核心卖点 | "一个 AI 帮你管库存、管订单、管客户，24 小时在线" |
| 引流手段 | 内容平台发布干货/案例 → 用户点链接进入小辉对话 |
| 转化方式 | 小辉现场演示 ERP/CRM 功能 → 开通试用 → 付费 |
| 定价模式 | SaaS 订阅 / 按模块收费 |

### 引擎二：商城平台增长

**卖的是商城的入驻和分销机会。**
目标：让同行业的人入驻商城开店、从公海选品上架、靠 AI 自动卖货赚钱。
手段：内容引流 → 小辉演示分销模式 → 入驻开店 → AI 帮成交。

| 维度 | 说明 |
|------|------|
| 产品 | 小辉智能分销商城（平台） |
| 目标用户 | 想开店但没货源的人、想做副业的个体、现有商户扩展线上渠道 |
| 核心卖点 | "不囤货、不发货、不管售后，AI 帮你卖货成交" |
| 引流手段 | 内容平台发布赚钱案例 → 用户点链接进入小辉 |
| 转化方式 | 小辉演示分销流程 → 当场帮你开店上架 → 开始卖 |
| 盈利模式 | 交易抽佣 / 分销分账 / 供货商服务费 |

### 两者的关系

```
用户看到"用 AI 管生意"的内容 → 进入小辉对话
  ├── 想管自己的生意 → 买小辉系统（引擎一成交）
  └── 想开店赚钱     → 入驻商城（引擎二成交）

已经买了小辉系统的客户 → 商城模块自带 → 同时也成为商城的分销商

商城分销商多了 → 交易量大了 → 需要更强大的管理系统 → 买小辉系统（引擎一反哺引擎二）
```

**飞轮效应**：系统用户 → 成为商城卖家 → 商城交易量增长 → 吸引更多卖家入驻 → 更多卖家需要系统 → 系统用户增长。

## 一、核心思路

**不是 AI 去追客户，是让客户主动找 AI——而且要让他们来了就不想走。**

两个引擎走同一套管道：
- 内容工厂（DeepSeek 自动生成）
- 全平台分发（微博/抖音/知乎/公众号/短视频）
- 统一入口（飞书/企微/微信/QQ 机器人链接 + 官网）
- 小辉自动接待（根据用户身份自动切换推销策略）
- 所有数据回写 DevLog，自动优化

```
DeepSeek 自动生成内容 → 定时分发到微博/抖音/知乎/公众号...
  → 用户看内容产生兴趣
  → 点击链接直接进入小辉对话（飞书/企微/微信/QQ 机器人）
  → 小辉自动接待、介绍、实时演示
  → 直接成交 或 无缝转人工
  → 所有数据回写 DevLog，自动优化下次内容
```

**小辉自己就是最好的 demo**。用户问"你能干嘛"，小辉当场调 API 演示——查库存、下订单、生成报表，不是放视频，是活系统在跑。

**入口就是现有的消息通道。** 不新建 App、不做小程序——飞书机器人链接、企微机器人链接、微信客服链接、QQ 机器人链接，这些就是营销入口。内容发到哪儿，链接跟到哪儿。

## 三、内容工厂（全自动）

### 3.1 内容生产流水线（全自动）

```
DeepSeek 根据行业热点 + 产品卖点 → 自动生成文案
  → 写入 DevLog 内容发布日历
  → Cron 定时触发 → 平台适配器自动发布
  → 监控评论/互动数据 → 自动回复评论区
  → 效果数据回写 DevLog → 优化下次生成策略
```

**人工可选介入点**：审核开关默认关（全自动），重要内容可选开启人工审核。

**双引擎内容矩阵**（同一套内容工厂，按目标分流）：

| 引擎 | 内容调性 | 典型标题 | 目标人群 | 发布平台 |
|------|---------|---------|---------|---------|
| 引擎一（系统） | 专业、效率、科技感 | "一个 AI 管三个人的活，老板终于能早回家了" | 中小企业老板 | 公众号/知乎/36氪 |
| 引擎一（系统） | 场景痛点 | "月底对账对到凌晨？小辉 30 秒拉完报表" | 财务/运营负责人 | 微博/抖音 |
| 引擎一（系统） | 客户案例 | "昆明螺蛳湾 XX 档口用后，库存差错从 15% 降到 0" | 同行业商户 | 公众号/官网 |
| 引擎二（商城） | 赚钱、自由、副业 | "不开店、不囤货、不发货，月入过万怎么做到的？" | 想做副业的人 | 抖音/小红书/微博 |
| 引擎二（商城） | 平台模式 | "商贸城 2000+ 商品任你选，上架就能卖，AI 帮你成交" | 现有商户 | 公众号/行业社群 |
| 引擎二（商城） | 分销案例 | "宝妈在家带娃，用小辉商城月销 3 万" | 个体/宝妈/学生 | 小红书/抖音 |

**AI 自动选题策略**：
- 监控各平台行业热搜关键词 → 自动匹配小辉能力 → 生成对应内容
- 引擎一关键词：库存管理、进销存、财务软件、数字化转型、批发管理
- 引擎二关键词：副业赚钱、零成本开店、一件代发、分销、在家赚钱

### 3.2 技术实现

**复用现有模块**：
- 内容生成：DeepSeek API（已在用）
- 发布调度：DevLog Cron 定时任务
- 平台连接：沿用飞书/企微的通道适配器模式

**新增**：平台连接器 —— 每个平台一个适配器（微信公众号 API / 知乎 API / 抖音开放平台等），统一接口。

## 四、对话转化（核心）

### 4.1 用户入口

不建新 App。入口就是已有的消息通道 + 官网：

**内容平台 → 消息通道链接**：
每条发布的内容底部附带：
- "想试试？点这里直接跟我聊 → 飞书机器人链接 / 企微机器人链接"
- 微博帖子底部、抖音简介、知乎回答末尾、公众号文章底部——统一放入口

**官网 → 嵌入式对话**：
网站底部直接嵌一个小辉对话窗口，用户不用跳转就能聊。

**所有入口统一路由**：不管从哪个平台、哪个链接进来，全部指向 `gateway/message`，由 OpenClaw 统一处理。小辉根据来源平台标记 `lead.source`，后续追踪都知道这个用户是从哪儿来的。

### 4.2 智能接待 + 自动分流

小辉的第一句话就要探测用户是冲着"管生意"还是"开店赚钱"来的。

```javascript
用户: "你好"
小辉: "你好！我是小辉。我能做两件事：
      ① 帮你管生意——库存/订单/客户/财务，一个 AI 全搞定。
      ② 帮你开店赚钱——不囤货不发货，AI 帮你卖货成交。
      你想了解哪个？"

// 用户选① → 引擎一：系统推销模式
用户: "管生意那个"
小辉: "你现在就可以试试。告诉我一个产品名，我帮你查库存、看成本、算利润。"
      // 实时调 /products → 返回库存 + 成本价 + 建议售价

// 用户选② → 引擎二：商城招商模式
用户: "开店赚钱"
小辉: "好的！我们商城有 2000+ 商品任你选，你不用进货、不管发货、不管售后。
      供货商统一处理，你只负责卖，AI 帮你成交。想先看看有哪些品类可以卖吗？"
      // 调 /mall/products → 返回公海商品列表
```

```
└──────────────────────────────────────────────┘
```

状态说明：
- **GREETING**: 小辉说开场白，探测用户意图
- **ENGINE_ONE / ENGINE_TWO**: 已确定推销策略
- **DEMOING**: 正在演示功能（可在此状态反复，切换不同 Demo）
- **TRIAL**: 引擎一 → 用户同意开通试用，调 `/auth/register-trial`
- **OPEN_SHOP**: 引擎二 → 用户同意开店，调 `/mall/merchant/register`
- **QUOTE**: 用户问价格 → 报价 → 可能成交或流失
- **TRANSFER**: 转人工 → 创建 CRM 任务 → 通知销售
- 任何状态都可以直接转 TRANSFER（用户说"让人来"）

## 八、关键决策

1. **全自动闭环** — 内容生成→发布→获客→演示→成交，全程无需人工干预
2. **入口即消息通道** — 不建 App，用飞书/企微/微信/QQ 机器人链接做入口
3. **官网做门面，对话做转化** — 官网展示品牌，所有"免费试用"按钮直接进入小辉对话
4. **对话即 Demo** — 小辉本身就是产品，现场调 API 演示，演示 = 产品体验
5. **只做内容引流，不主动私信** — 合规、不封号、用户主动意向质量高
6. **复用现有模块** — CRM/报价/DevLog/Auth/Printer 全线复用，不另起炉灶
7. **无声转人工** — 转接时对话上下文自动同步给销售，用户不用重复说
8. **数据闭环优化** — 所有漏斗数据回写 DevLog，自动优化内容策略和转化路径
